4//! Low-level, `unsafe` bindings to the C APIs of [cFE](https://github.com/nasa/cFE)
5//! and [OSAL](https://github.com/nasa/osal).
6//!
7//! This is `n2o4`'s equivalent of a
8//! [`*-sys` crate](https://doc.rust-lang.org/cargo/reference/build-scripts.html#-sys-packages)
9//! for the cFE and OSAL APIs.
10//! Generally, users of `n2o4` will not use this module directly,
11//! preferring the safe wrappers provided in the rest of `n2o4`.
12//! However, this module is available
13//! if one needs to use something for which safe wrappers haven't been made.
14//!
15//! The definitions in this module are auto-generated by
16//! [bindgen](https://github.com/rust-lang/rust-bindgen),
17//! so there's no Rustdoc comments available.
18//! Check out the
19//! [cFE user's guide](https://github.com/nasa/cFE/blob/gh-pages/cfe-usersguide.pdf)
20//! for details about types and functions.
21//!
22//! Those identifiers starting with `S_`, `X_`, and `SHIM_` are the result
23//! of hand-written C code required to make `bindgen` create bindings for
24//! certain constants and function; each is the same as the C constant
25//! or function without the prefix in its name.
26
